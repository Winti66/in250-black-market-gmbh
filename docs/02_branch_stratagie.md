#Die Branch Stratagie con Black Market Software GmBh

##Der Main Branch
Im Main Branch darf nie gearbeitet werden da dies das Orginal ist.


##Feature Branch
Für jede änderung der Software musss ein neuer Branch erstellt werden.
Benennung der Branch "feat/Name der Änderung oder bei einem bugfix/Name des Fehlers


##Pull Requests
Für jede änderung muss ein Pull Requests erstellt werden. Diese werden nachher von einem Mitarbei der Black Market Software GmBh überprüft bevor sie in das Main gemerge werden.


##Release
Gut optimierung der Software im Main werden mit Git-Tas (z.bs. v1.1) gekennzeichnet
